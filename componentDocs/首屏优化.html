    # 通过理解css加载来优化首屏加载

    来看看下面的渲染流程

    //theme.css
div{     
    color : coral;    
    background-color:black
}

<html>
    <head>    
        <link href="theme.css" rel="stylesheet">
    </head>
    <body>    
        <div>geekbang com</div>
    </body>
</html>

    <img src="/browser/流水1.webp" width="800" height="500>

    渲染进程来解析 HTML 数据并构建 DOM。 请求 HTML 数据和构建 DOM 树是有一段空闲时间，这个时间可能成为页面渲染的瓶颈。

    渲染进程在接收到 HTML 字节流的时候，会开启一个预解析线程，遇到 js 文件或 css 文件，预解析线程会提前下载这些数据。

    预解析线程会提前下载 theme.css 文件。还有一个空闲时间就是构建 DOM 树结束之后，theme.css 文件还未下载完，可能有一个空闲阶段，这个时间段渲染流水线无事可做，下一步是合成布局树。合成布局树需要 CSSOM 和 DOM。

    # CSSOM 的作用

    • 提供给 js 操作样式表的能力。
    • 为布局树的合成提供基础的样式信息. CSSOM 体现在 document.styleSheets. (布局树的结构基本就是复制 DOM 树的结构，只是会过滤掉一些无用标签)。

    # 复杂场景

    //theme.css
div{     
    color : coral;    
    background-color:black
}

<html>
    <head>    
        <link href="theme.css" rel="stylesheet">
    </head>
    <body>    
        <div>geekbang com</div>
        <script>        
            console.log('time.geekbang.org')    
        </script>
        <div>geekbang com</div>
    </body>
</html>

    <img src="/browser/流水2.webp" width="800" height="500>

    遇到 js，就需要暂停 DOM 解析去执行 js。js可能会修改当前状态的 DOM。

    但是在执行 js 之前，如果包含外部 css 文件的引用，渲染引擎需要将这些内容转为 CSSOM。 js 有 修改 CSSOM 的能力，所以在执行 js 之前，需要依赖 CSSOM。

    //theme.css
div{     
    color : coral;    
    background-color:black
}

//foo.js
console.log('time.geekbang.org')

<html>
    <head>    
        <link href="theme.css" rel="stylesheet">
    </head>
    <body>    
        <div>geekbang com</div>
        <script src='foo.js'></script>
        <div>geekbang com</div>
    </body>
</html>

    <img src="/browser/流水3.webp" width="800" height="500>

    HTML 预解析器识别出 CSS 文件和 js 文件需要下载，同时发起着两个文件的下载请求。这两个文件是重叠下载的。

    # 影响页面展示的因素

    渲染流水线影响到了首次页面的展示，而首次页面展示的速度直接影响到用户的体验。

    从发起 URL 开始，到首次显示页面的内容，在视觉上经历了三个阶段

    • 第一个阶段, 等请求发出去之后，到提交数据阶段。是之前的页面。
    • 第二个阶段，从提交数据之后渲染进程会创建一个空白页，称为解析白屏，并等待 CSS 文件和js 文件的加载完成，生成 CSSOM和DOM，然后合成布局树，最后要经过一系列步骤准备首次渲染。
    • 第三个阶段，等首次渲染完成之后，就开始进入完整页面的生成阶段，然后页面会一点点被绘制出来。

    通过情况下的瓶颈体现在下载CSS文件、下载JS文件和执行JS。

    # 优化策略

    • 通过内联 js、内联css来移除下载。
    • 减少文件大小。webpack。
    • 将一些不需要解析 HTML 阶段使用的 js 标记 sync defer。
    • 对于大的 CSS 文件，可以拆分为多个不同用途的 CSS 文件。
