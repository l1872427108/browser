    # 内存泄露

    # 什么是内存泄露

    程序的运行需要内存，只要保存东西就需要分配内存。当不再需要时需要及时的释放掉不再使用的内存，否则，内存占用就会越来越高，轻则影响系统性能，重则导致进程崩溃。

    # js 内存管理

    垃圾回收章节

    # 内存泄漏

    在理解了引擎如何进行标记内存不可达的，我们才能明白什么时候回收。我们来看看哪些东西本应该被回收，但却因为可达而留在了内存当中。

    # 意外的全局变量

    function foo() {
    a = 1;
}
foo()

    即使foo执行完成，本应该释放掉，但却在 window 上绑定了一个属性 a。从可达性的算法来说，遍历 root，只要 window 存在，a 属性就不会被释放。

    虽然只是一个简单的字符，占用的内存空间不多，但这确确实实是应该释放掉的一块空间，这样长时间存在与内存当中。

    下面也是一样的道理。

    function foo() {
    this.a = 1;
}
foo();

    # 定时器或回调函数

    setTimeout(() => {
    // 回调
}, 1000);

    回到函数，即使没有与之关联的，按道理来说就应该释放掉了，但是因为计时器回调函数不得已释放，只有计时器停止才会被回收。 明确的移除它们是很重要的。

    # 事件

    var element = document.getElementById();
element.addEventListener('click', () => {
    element.innerHTML = 'xx';
})

    一旦不在需要，明确的移除。

    # 闭包

    闭包是非常有用的，但是滥用闭包会导致内存泄漏，因为外层遍历一直被引用着，释放不掉，一个还好，太多的话就会占用大量的内存。

    我们可以明确的给它赋值 null; 来断开
