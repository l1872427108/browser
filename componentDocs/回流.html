    # 回流

    也就做重排。 在第一个渲染之后，会经过一整个关键渲染路径。如果再次经过布局，这个就叫做回流。

    # 触发条件

    对 DOM 结构的修改引发 DOM 几何尺寸的变化的时候，会发生回流的过程。

    具体的能够触发回流：

    1. 一个 DOM 元素的几何属性变化，常见的有 width、height、padding、margin、left、top、border 等。

    2. 使 DOM 节点发生 增减或移动。

    3. 读写 offset ， scroll， client 这些属性的时候，浏览器为了获取这些值，会进行回流操作。

    4. 调用 window.getComputedStyle 方法。

    # 回流路径

    如果触发了回流，DOM结构发生变化，则重新渲染 DOM 树，然后需要从 生成 DOM 树开始全部再走一遍。开销非常的大。

    # 重绘

    是一个元素外观的改变所触发的浏览器行为。 改变背景、颜色、outline 。呈现一个新的外观.外观、阴影、颜色、边框.

    重绘是以图层为单位的。 如果图层中某个元素需要重绘，整个图层都需要绘制。

    # 重绘路径

    会进行样式计算，但是会跳过生成布局树和构建图层树的阶段，直接生成绘制列表，然后进行一系列的操作。

    # GPU 加速

    是直接合成的，比如使用 CSS3 的 transform、opacity、filter 这些属性可以实现合成的效果。

    # GPU加速的原因

    在合成情况下，会直接进入提交合成线程处理。

    1. 能够充分发挥 GPU 的优势。合成线程生成位图的过程中会调用线程池，并在其中使用 GPU 进行加速生成。
    2. 没有占用主线程的资源，即使主线程卡住了，效果依赖能够流畅的展示。 (合成线程)

    # 优化方案

    1. 元素变化， 定位操作位置。 使用 css3 的 transform
    2. 不要使用 table 布局.
    3. 将多次改变样式属性操作合并成一次行操作。 class
    4. 将 DOM 离线后操作。 display: none; 或者隐藏掉，不再页面上显示。
    5. 文档碎片。createDocumentFragment 进行批量的 DOM 操作。
    6. 对于 resize、scroll 等进行防抖/节流处理。
    7. 添加 will-change: transform, 让渲染引擎为其单独实现一个图层，当这些变换发生时，仅仅是利用合成线程去处理这些变换，而不牵扯到主线程，大大提高渲染效率。这个变化不限于 transform, 任何可以实现合成效果的 CSS 属性都适用 will-change 来声明。
