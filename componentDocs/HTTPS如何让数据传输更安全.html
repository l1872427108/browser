# HTTPS如何让数据传输更安全

HTTP 是明文传输的，因此在传输的每一个环节，数据都有可能被第三方窃取或者串改，HTTP 数据经过 TCP 层，然后经过 WIFI 路由器、运营商 和 目标服务器，这些环节都是有可能被篡改的。

HTTPS 是加强版的 HTTP，在 HTTP 和 TCP 之间建立了一个中间层，当 HTTP 和 TCP 通信不再想之前那样直接通信，而是经过一个中间层进行加密，将加密后的数据包传给 TCP。数据传回时，TCP必须将数据包解密，才能传给上面的 HTTP。

对称加密：加密和解密都是使用同样的密钥。

非对称加密：如果 A 加密过的数据包只能用 B 解密，反之，B 加密的数据包只能用 A 解密。

# 加解密过程

  1. 浏览器会给服务器发送一个随机数 client_random 和 一个加密的方法列表。
  2. 服务器接收后给浏览器返回另一个随机数 server_random 和加密方法。
  3. 现在两者都有 client_random 和 server_random 和加密方法。
  4. 用这个加密方法将两个随机数混合起来生成密钥，就是通信的密码了。

# 应用

  1. 对称加密之应用 如果使用对称加密，那么第三方可以在中间获取到 client_random、server_random 和加密方法，由于这个加密方法同时可以解密，所以中间人可以对进行解密，拿到数据。

  2. 非对称加密之应用 服务器手里有一个公钥，有一个私钥。私钥是保存在服务器的。

  • 浏览器把 client_random 和加密方法列表传过来，服务器接收到，把 server_random、加密方法 和 公钥传给服务器。
  • 然后浏览器用公钥将 client_random 和 server_random 加密，生成与服务器通信的暗号。
  • 公钥加密的只能用私钥来解，所以中间人就算截取到，但是没有私钥也是不能够解密的。因此能够保障数据的安全。

但是服务器的数据只能用私钥进行加密，中间人拿到公钥，就可以对服务器传来的数据进行解密了。

# 对称和非对称结合

  1. 浏览器向服务器发送 client_random 和加密方法。
  2. 服务器收到，返回 server_random, 加密方法和公钥。
  3. 浏览器接收到，生成另一个随机数 pre_random, 并且用公钥加密，传给服务器。
  4. 服务器用私钥解密这个被加密后的 pre_random.
  5. 然后两者用相同的加密方法混合这三个随机数。生成密钥。
