# DOM和VNode的不同

# DOM 的缺陷

通过 js 来操纵 DOM 会影响到整个渲染流水线。

对于简单的页面来说，DOM 结构比较简单，操作 DOM 对用户的体验不会产生太大的影响。但是对于复杂的页面或者非常火的单页应用来说，DOM 结构是非常复杂的。需要不断的修改 DOM。都会引起重排、重绘或者合成操作。

# 虚拟DOM

  • 页面改变的内容应用到虚拟 DOM 上，而不是直接应用到 DOM上。
  • 虚拟 DOM 不会直接去渲染页面，而是进行比对找到需要改动的部分。
  • 在虚拟 DOM 收集到足够的改变，才会一次性的应用到真实的 DOM。

# 从双缓存视角来看虚拟DOM

屏幕是从前缓冲区读取数据然后显示，比如一整个页面，需要多次才能完成。如果直接放入前缓冲区，就会一部分一部分显示出来，让用户感到页面的闪烁。常用来解决无效刷新和闪屏问题。

使用双缓存，可以将计算的中间结果存放到另一个缓冲区中，等全部计算完成，在一次性的复制到前缓冲区。

虚拟DOM 也是会在完成一次性操作之后，再将结果应用到 DOM, 减少不必要的更新。

# MVC模式看虚拟DOM

我们把虚拟DOM看作是 MVC 的视图部分。控制器和模型都是由Vue提供。

  • 监控到 DOM 变化，一旦 DOM 发生变化，控制器就会通知模型，让其更新数据。
  • 模型数据更新好之后，根据数据来生成新的虚拟DOM。
  • 比对。触发DOM节点更新。
  • 一次性触发渲染流水线。

前端框架，离不开设计模式。基于MVC的设计思想广泛的渗透到各种场合，并且基于MVC衍生出很多其他模式（MVP、MVVM等）。
