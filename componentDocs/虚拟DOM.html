# 虚拟DOM

  1. 什么是虚拟DOM? 虚拟 DOM 就是用一个 JS 对象来描述一个 DOM 节点。
<p class="wrap">我是小明</p>
{
    tag: 'p',
    attrs: {
        class: 'wrap'
    },
    text: '我是小明',
    children: []
}

我们就是使用上面的方式，将一个 DOM 节点必要的东西用 js 来表示出来。这个对象表示的对象就是真实 DOM 节点的虚拟 DOM 节点。 （DOM 树是非常相似的）。

  1. 为什么要有虚拟 DOM Vue 是数据驱动视图的，数据变化对应的视图就要随之更新。但是更新 DOM 是非常操作时非常耗时的。通过下面的演示我们看出来一个简单的 DOM 标签就已经非常复杂了，何况是操作一个复杂的页面。
let div = document.createElement('div');
let str = '';
for (const key in div) {
    str += key + ''
}
console.log(str);

更需要新视图，操作DOM是必不可少的，那么有什么办法能够尽量少的操作 DOM。

尽量不要盲目的更新视图，而是通过比对数据变化前后的状态，计算出哪些地方需要更新，只需要更新需要更新的地方。

相当于使用 js 的计算性能来换取操作 DOM 的性能。这个比对的过程就是 DOM-Diff 的过程。

# VNode哪个时候产生

是发生在编译阶段，将模版字符串，解析成 render 函数。调用 render 函数，就会返回一个虚拟DOM（VNode）。警告一系列的对比之后，将 VNode 绘制到页面上。

我么你来看 Vue.prototype._update , 在文件 src/core/instance/lifecycle.js 文件中。

Vue.prototype._update = function (vnode) {
     const vm: Component = this
     const prevVnode = vm._vnode
     vm._vnode = vnode
     if (!prevVnode) {
      // initial render
      vm.$el = vm.__patch__(vm.$el, vnode, hydrating, false /* removeOnly */)
    } else {
      // updates
      vm.$el = vm.__patch__(prevVnode, vnode)
    }
}

vm.patch. 就是元素渲染、VNode 做 diff 并修改元素、销毁元素的地方。

prevVnode 是指旧的 VNode，第一次初始化过程中，旧的VNode是不存在的，所以自然不存在diff的过程， 会创建根据 VNode 直接绘制到页面中。

当数据更新再调用 _update 方法时候，prevVnode 是旧的 vNode。此时就传入 vm.patch(prevVnode, vnode) 新旧 VNode 进行 diff。做相应的更改在展示出来。

至于什么时候会调用 _update 方法，watcher 提到的每一个 Watcher 都保存了一个 update 更新视图的方法。 watcher 对象更新时会调用 updateComponent, 今儿调用 vm._update 方法，vm._render()

接下来进入diff算法。

# 项目结构

# VNode

VNode 构造函数在 src/core/vdom/vnode.js 中，该文件主要用来定义了 VNode 对象包含的类型有哪些。并且也定义了几个创建 VNode 对象的方法。

export default class VNode {
  // 构造函数
  constructor (
    tag?: string,
    data?: VNodeData,
    children?: ?Array<VNode>,
    text?: string,
    elm?: Node,
    context?: Component,
    componentOptions?: VNodeComponentOptions,
  ) {
    // 当前节点的标签名
    this.tag = tag
    // 当前节点对应的对象，包含了具体的一些数据信息
    this.data = data
    // 当前节点的子节点，是一个数组
    this.children = children
    // 当前节点的文本
    this.text = text
    // 当前虚拟节点对应的真实 dom 节点
    this.elm = elm
    // 当前节点的名字空间
    this.ns = undefined
    // 当前组件节点对应的 Vue 实例
    this.context = context
    // 函数式组件对应的 Vue 实例
    this.fnContext = undefined
    this.fnOptions = undefined
    this.fnScopeId = undefined
    // 节点的 key 属性，被当作节点的标志，用以优化
    this.key = data && data.key
    // 自定义组件上部分组件属性
    this.componentOptions = componentOptions
    // 当前节点对应的组件的实例
    this.componentInstance = undefined
    // 当前节点的父节点
    this.parent = undefined
    // 是否为原生 HTML 或只是普通文本。
    this.raw = false
    // 静态节点标志
    this.isStatic = false
    // 是否作为根节点标志
    this.isRootInsert = true
    // 是否为注释节点
    this.isComment = false
    // 是否为克隆节点
    this.isCloned = false
    // 是否有 v-once 指令
    this.isOnce = false
  }

  // DEPRECATED: alias for componentInstance for backwards compat.
  /* istanbul ignore next */
  get child (): Component | void {
    return this.componentInstance
  }
}

构造函数可以接收的参数有七个，分别是 tag标签名、data节点相关数据、children子结点对象数组、text文本内容、elm原生结点元素、context指当前元素所在的Vue实例、componentOptions保存自定义组件上部分组件属性。

通过属性之间的不同搭配，就可以创建出来描述不同类型节点的 DOM。

  // 唯一不同就是新节点的 isClone 为 true

  return cloned
}

# DOM-Diff

在 Vue 中，把 DOM-Diff 的过程叫做 patch（补丁） 过程。也就是对比新旧两份 VNode。 旧的 VNode （oldVNode）是数据变化之前的虚拟DOM节点，新的 VNode（newVNode） 是数据变化之后将要渲染到视图上的虚拟DOM节点。我们要有一个清除的认知，那就是要以 newVNode 为基准，改造 oldVNode ，这就是 patch 的过程。

其实节点的操作无非是创建节点、删除节点、更新节点三种操作。

  • 创建节点：newVNode 有而 oldVNode 中没有， oldVNode 中创建.
  • 删除节点：newVNode 没而 oldVNode 中有，oldVNode 中删除。
  • 更新节点：newVNode 和 oldVNode 都有，以 newVNode 为基准，更新 oldVNode.

# 创建节点 (newVNode有，而oldVNode没)

对于 VNode 能够被创建的节点有三类，元素节点、文本节点、注释节点。

// src/core/vdom/patch.js

function createElm (
    vnode,
    insertedVnodeQueue,
    parentElm,
    refElm,
    nested,
    ownerArray,
    index
  ) {
    const data = vnode.data
    const children = vnode.children
    const tag = vnode.tag
    if (isDef(tag)) {
      vnode.elm = nodeOps.createElementNS(vnode.ns, tag)
      createChildren(vnode, children, insertedVnodeQueue)
      insert(parentElm, vnode.elm, refElm)
    } else if (isTrue(vnode.isComment)) {
      vnode.elm = nodeOps.createComment(vnode.text)
      insert(parentElm, vnode.elm, refElm)
    } else {
      vnode.elm = nodeOps.createTextNode(vnode.text)
      insert(parentElm, vnode.elm, refElm)
    }
  }
  • createElm 方法通过判断该 VNode 节点是否有 tag 标签，如果有 tag 属性就认为是元素节点。通过调用 createElementNS 来创建元素节点，还会遍历所有的子节点，创建好之后 insert 插入到当前元素节点里面。
  • 如果是注释节点，只需要判断 isComment 属性即可。
  • 如果都不是，那么就是文本节点。

# 更新节点

更新节点发生在 newVNode 和 oldVNode 中都存在时，通过比对来进行更新。

  1. 如果 newVNode 和 oldVNode 都是静态节点. 静态节点就是不包含任何的变量，通俗点说就是死数据，不会改变的就是静态节点, 判断是否为静态，发生在模版字符解析阶段，是一个很大的优化手段，避免了许多不必要的计算。 这种情况下直接跳过，不需要比对。

  2. 如果 newVNode 是文本节点. 需要看 oldVNode 是否也是文本节点。如果是，看文本是否相同，如果不同把 oldVNode 的文本改成和 newVNode 一样的文本。如果 oldVNode 不是文本节点，直接调用 setTextNode 改成和 newVNode 一样的文本节点。

  3. 如果 newVNode 是元素节点

    • 该节点包含子节点 查看旧的节点是否包含子节点，如果包含子节点需要递归比对更新子节点。 如果不包含子节点，需要创建一份和 newVNode 一样的子节点插入到节点里面。

    • 该节点不包含子节点 不包含子节点，也不是文本节点，就说明是一个空节点，不管里面是啥，直接清空。

// src/core/vdom/patch.js
function patchVnode (
    oldVnode,
    vnode,
    insertedVnodeQueue,
    ownerArray,
    index,
    removeOnly
  ) {
    // vnode 和 oldVNode 是否完全一样，一样，就退出程序。
    if (oldVnode === vnode) {
      return
    }

    // vnode 和 oldVNode 是否是静态节点，是，就退出程序
    if (isTrue(vnode.isStatic) &&
      isTrue(oldVnode.isStatic) &&
      vnode.key === oldVnode.key &&
      (isTrue(vnode.isCloned) || isTrue(vnode.isOnce))
    ) {
      vnode.componentInstance = oldVnode.componentInstance
      return
    }

    let i
    const data = vnode.data
    if (isDef(data) && isDef(i = data.hook) && isDef(i = i.prepatch)) {
      i(oldVnode, vnode)
    }

    // vnode 没有 text 属性
    if (isUndef(vnode.text)) {
      // vnode 的子节点与 oldVnode 的子节点是否都存在
      if (isDef(oldCh) && isDef(ch)) {
        // 若都存在，判断子节点是否相同，不同则更新子节点
        if (oldCh !== ch) updateChildren(elm, oldCh, ch, insertedVnodeQueue, removeOnly)

      // 只有 vnode 的子节点存在
      } else if (isDef(ch)) {
        if (process.env.NODE_ENV !== 'production') {
          checkDuplicateKeys(ch)
        }
        // 判断 oldVNode 是否有文本。
        // 若没有 则把vnode的子节点添加到真实DOM中
        // 若有，则清空Dom中的文本，再把vnode的子节点添加到真实DOM中
        if (isDef(oldVnode.text)) nodeOps.setTextContent(elm, '')
        addVnodes(elm, null, ch, 0, ch.length - 1, insertedVnodeQueue)

      // 若只有oldnode的子节点存在
      } else if (isDef(oldCh)) {
        // 清空 DOM 中的子节点
        removeVnodes(oldCh, 0, oldCh.length - 1)

      // 若vnode和oldnode都没有子节点，但是oldnode中有文本
      } else if (isDef(oldVnode.text)) {
        // 清空oldnode文本
        nodeOps.setTextContent(elm, '')
      }

    // 如有text，判断 oldVNode 与 oldVNode 属性是否相同。
    } else if (oldVnode.text !== vnode.text) {
      // 若不相同，则用 vnode 的 text 替换真实 DOM 的文本。
      nodeOps.setTextContent(elm, vnode.text)
    }
  }

如果 newVNode 和 oldVNode 都包含子节点，就会调用 updateChildren 方法，来对比更新子节点。

接下来就是 diff 算法的重中之重了。

# updateChildren

更新子节点可能发生的操作：创建子节点、删除子节点、移动子节点、更新节点。

  1. 创建子节点 如果 newVNode 的子节点没有在 oldVNode 的子节点里面找到，就需要创建这个子节点。创建节点和前面使用的方法是一样的，重点是创建好的节点要插入到哪个位置。

  2. 删除子节点 通过将 newVNode 的子节点都找一遍，能在 oldVNode 中找到旧处理它，如果找过一次之后，发现 oldVNode 中还有未处理的子节点，说明这些节点是不需要的，旧将节点删除。

  3. 更新子节点 如果 newVNode 中的某个子节点，在 oldVNode 中找到了相同的子节点，所处的位置也是一样的，就需要更新该节点了。

  4. 移动子节点 如果 newVNode 里的子节点在 oldVNode 中找到了相同的子节点，但是位置不同，就需要以 newVNode 中子节点的位置，去移动 oldVNode 中子节点的位置。

我们来看看 updateChildren 方面干得事情。 Vue 意识到如果只是存粹的 newVNode 和 oldVNode 两两比较，这样循环算法的时间复杂度很大，不利于性能提升。所以一开始是通过比较特殊位置节点的。

// src/core/vdom/patch.js
function updateChildren (parentElm, oldCh, newCh, insertedVnodeQueue, removeOnly) {
    let oldStartIdx = 0  // oldChildren开始索引
    let oldEndIdx = oldCh.length - 1  // oldChildren结束索引
    let oldStartVnode = oldCh[0]  // oldChildren中所有未处理节点中的第一个
    let oldEndVnode = oldCh[oldEndIdx]  // oldChildren中所有未处理节点中的最后一个


    let newStartIdx = 0  // newChildren开始索引
    let newEndIdx = newCh.length - 1  // newChildren结束索引
    let newStartVnode = newCh[0]  // newChildren中所有未处理节点中的第一个
    let newEndVnode = newCh[newEndIdx]  // newChildren中所有未处理节点中的最后一个

    let oldKeyToIdx, idxInOld, vnodeToMove, refElm

    // removeOnly is a special flag used only by <transition-group>
    // to ensure removed elements stay in correct relative positions
    // during leaving transitions
    const canMove = !removeOnly

    // 以"新前"、"新后"、"旧前"、"旧后"的方式开始比对节点
    while (oldStartIdx <= oldEndIdx && newStartIdx <= newEndIdx) {
      // oldStartVnode 不存在
      if (isUndef(oldStartVnode)) {
        oldStartVnode = oldCh[++oldStartIdx] // Vnode has been moved left
      } else if (isUndef(oldEndVnode)) {
        // oldEndVnode 不存在
        oldEndVnode = oldCh[--oldEndIdx]
      } else if (sameVnode(oldStartVnode, newStartVnode)) {
        // 新前与旧前节点相同，进行patch更新
        patchVnode(oldStartVnode, newStartVnode, insertedVnodeQueue, newCh, newStartIdx)
        oldStartVnode = oldCh[++oldStartIdx]
        newStartVnode = newCh[++newStartIdx]
      } else if (sameVnode(oldEndVnode, newEndVnode)) {
        // 新后与旧后节点相同
        patchVnode(oldEndVnode, newEndVnode, insertedVnodeQueue, newCh, newEndIdx)
        // oldEndIdx和newEndIdx都减1, 前移一个位置
        oldEndVnode = oldCh[--oldEndIdx]
        newEndVnode = newCh[--newEndIdx]
      } else if (sameVnode(oldStartVnode, newEndVnode)) { // Vnode moved right
        // 新后与旧前节点相同, 两个节点进行patch更新
        patchVnode(oldStartVnode, newEndVnode, insertedVnodeQueue, newCh, newEndIdx)
        canMove && nodeOps.insertBefore(parentElm, oldStartVnode.elm, nodeOps.nextSibling(oldEndVnode.elm))
        oldStartVnode = oldCh[++oldStartIdx]
        newEndVnode = newCh[--newEndIdx]
      } else if (sameVnode(oldEndVnode, newStartVnode)) { // Vnode moved left
        // 新前与旧后节点相同, 两个节点进行patch更新
        patchVnode(oldEndVnode, newStartVnode, insertedVnodeQueue, newCh, newStartIdx)
        canMove && nodeOps.insertBefore(parentElm, oldEndVnode.elm, oldStartVnode.elm)
        oldEndVnode = oldCh[--oldEndIdx]
        newStartVnode = newCh[++newStartIdx]
      } else {
        // 以key为键，索引值为value，生成新的对象oldKeyToIdx。
        if (isUndef(oldKeyToIdx)) oldKeyToIdx = createKeyToOldIdx(oldCh, oldStartIdx, oldEndIdx)

        //  查询newStartVnode是否有key值，并查找oldKeyToIdx是否有相同的key
        idxInOld = isDef(newStartVnode.key)
          ? oldKeyToIdx[newStartVnode.key]
          : findIdxInOld(newStartVnode, oldCh, oldStartIdx, oldEndIdx)

        // 如果newStartVnode没有key或oldKeyToIdx没有相同的key
        if (isUndef(idxInOld)) { // New element
          // 新增节点并插入到合适位置  (合适的位置就是在 oldChildren 里面的未处理节点之前)
          createElm(newStartVnode, insertedVnodeQueue, parentElm, oldStartVnode.elm, false, newCh, newStartIdx)
        } else {
          // 如果在 oldChildren 里找到了当前循环的 newChildren 里的子节点
          vnodeToMove = oldCh[idxInOld]

          if (sameVnode(vnodeToMove, newStartVnode)) {
            patchVnode(vnodeToMove, newStartVnode, insertedVnodeQueue, newCh, newStartIdx)
            oldCh[idxInOld] = undefined
            canMove && nodeOps.insertBefore(parentElm, vnodeToMove.elm, oldStartVnode.elm)
          } else {
            // same key but different element. treat as new element
            createElm(newStartVnode, insertedVnodeQueue, parentElm, oldStartVnode.elm, false, newCh, newStartIdx)
          }
        }
        newStartVnode = newCh[++newStartIdx]
      }
    }

sameVnode 方法判断两个 vnode 可不可以复用为一个节点。频繁的创建删除 dom 元素，会造成性能的浪费。更好的方法就是尽可能的复用原有的 dom。若两个 vnode 的 key 相同，tag相同，或者都是注释，都有data。如果是input则要data和attrs都有或都没有，且type必须相同。 如果是我们自定义的组件，一般情况下都包裹在一个根元素中，sameVnode 比较两个 vnode 对象返回的是true，所以直接会 patchVnode。

// src/core/vdom/patch.js
function sameVnode (a, b) {
  return (
    // 判断 key 的值是否相等
    a.key === b.key &&
    a.asyncFactory === b.asyncFactory && (
      (
        a.tag === b.tag &&
        a.isComment === b.isComment &&
        isDef(a.data) === isDef(b.data) &&
        sameInputType(a, b) 
      ) || (
        isTrue(a.isAsyncPlaceholder) &&
        isUndef(b.asyncFactory.error)
      )
    )
  )
}

oldStartIdx、newStartIdx、oldEndIdx、newEndIdx 都是指针。整个过程，就是不断的移动指针。

oldStartVnode、newStartVnode、oldEndVnode、newEndVnode 与上面的指针对应，指向 VNode 节点。
