# XSS 攻击

XSS 全称 Cross Site Scripting(跨站脚本), 为了和 CSS 区分，才叫做 XSS。XSS 攻击时指浏览器中执行恶意脚本（无论是跨域还是同域），从而拿到用户的信息并进行操作。

# 可以完成的操作

  1. 窃取 Cookie.
  2. 监听用户行为，比如输入账号密码后直接发送到黑客服务器。
  3. 修改 DOM 伪造登录表单.
  4. 在页面中生成浮动窗口广告。

XSS 攻击实现有三种方式 --- 存储型、反射型 和 文档型。

# 存储型

存储型，就是将恶意脚本存储起来，存储型的 XSS 将脚本存储到服务端的数据库，然后在客户端执行这些脚本，从而达到攻击的效果。

常见的场景是留言评论区提交一段脚本代码，如果前后端没有做好转义工作，那评论就会拿到数据库，在页面渲染的过程中直接执行一段未知的逻辑，是非常恐怖的。

# 反射型

反射型 XSS 指的是恶意脚本作为网络请求的一部分。

http://puge.cn?q=

服务器拿到 q 参数，然后返回给浏览器，浏览器将这些内容作为 html 的一部分解析。发现是一个脚本，直接执行，就被攻击了。

恶意脚本是通过作为网络请求的参数，经过服务器，然后再反射到 HTML 文档中，执行解析。服务器是不会存储这些恶意代码的。

# 文档型

文档型的 XSS 攻击不会经过 服务端，而是作为中间人的角色，在数据传输过程劫持到网络数据包，然后修改里面的 html 文档。

这样的劫持方式包括： WIFI 路由劫持 或者 本地恶意软件 等.

# 措施

XSS 就是让恶意脚本在浏览器中执行。

  1. 千万不能够相信任何用户的输入！

无论在前端和服务端，都要对用户的输入进行转码或者过滤。

转码之后的代码在 html 解析的过程中时无法执行的。

也可以把 script 标签给删除掉。

  1. 利用 CSP CSP 是浏览器的内容安全策略，核心思想就是服务器来决定浏览器加载哪些资源。
  • 限制其他域下的资源加载。
  • 禁止向其他域提交数据。
  • 提供上报机制，帮助及时发现 XSS 攻击。
  1. 利用 HTTPOnly 很多 XSS 攻击脚本都是用来读取 Cookie 的，而设置 Cookie 的 HttpOnly 属性后， js 便无法读取 Cookie 的值，很好的防御 XSS 攻击。
