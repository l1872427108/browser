# 浏览器缓存

浏览器缓存可以分为两种：一种是需要发送 HTTP 请求，一种不需要发送.

# 强缓存

检查强缓存，不需要进行发送 HTTP 请求。 在相应的字段来进行检查。

在 HTTP/1.0 和 HTTP/1.1 ，强缓存的字段是不一样的。

在 HTTP/1.0 阶段，使用的是 Expires 字段。在 HTTP/1.1 阶段使用 Cache-Control 字段.

# Expires

Expires 存在于服务端返回的响应头中，告诉浏览器在这个过期时间之间可以从缓存中去获取数据，不用在此请求。

Expires: Wed, 22 Nov 2019 09:41:00 GMT

表示资源将在 xxxx年xxxx月xxxx日资源过期。

但是这种存在一个问题就是：对于客户端和服务器的时间是不一致的，服务器和浏览器的时间并不一致。所以在 HTTP/1.1 中就抛弃了。

# Cache-Control

Cache-Control 是不一样的，它并没有像 Expires 一样设置的是时间点，而是通过时长来控制的。

Cache-Control:max-age=3600

则会在 3600 秒之后过期，也就是 1 个小时内过期。

并且 Cache-Control 上有很多的属性的，可以组成多种形式，完成多种场景的判断。

  1. public：对于客户端和代理服务器都可以设置缓存。 因为一个请求，可能经过多个代理服务器才能够到达服务器，不管是浏览器可以被缓存，任何一个代理服务器都可以被缓存。
  2. private: 能够缓存的就只有是浏览器了，而没有代理服务器。
  3. no-cache: 跳过强缓存，进行 HTTP 请求，进入协商缓存阶段。
  4. no-store: 不会进行任何形式的缓存。
  5. s-maxage: 是针对于代理服务器的缓存时间段。

# 强缓存过期

当强缓存过期之后，就会进入到协商缓存阶段。

# 协商缓存

当强缓存过期之后，浏览器就会携带上缓存tag，发送 HTTP，服务器接收到之后，会根据缓存 tag 去决定是否使用缓存，这就是协商缓存。

# Last-Modified

代表的是最后的修改时间，浏览器在请求服务器之后，服务器的响应就会带上 Last-Modified 这个字段。

当下一次浏览器需要请求头携带 If-Modified-Since 这个字段，这个字段的值就是浏览器返回的最后修改时间。

浏览器拿到这个值后，会与该资源的最后修改时间会进行比较：

  1. 如果请求头中的时间是小于最后修改的时间，代表是时候进行更新了，就会像正常的 HTTP 请求一样去返回响应。
  2. 否则返回 304，告诉浏览器直接去缓存拿取资源。

# ETag

服务器会根据资源内容生成一个唯一标识，只要内容不发送变化，这个标识就不会改变。

浏览器在请求的时候，会携带这个 If-None-Match 这个字段内容。

服务器拿到这个字段之后，就会与资源的 ETag 进行比较：

  1. 如果两者不一样，代表要进行跟新了，跟正常HTTP响应一样，返回响应。
  2. 否则返回 304，告诉浏览器去直接拿取缓存。

# Last-Modified 和 ETag

都是进行协商缓存的。

精度上： Last-Modified：

  1. 是针对于内容修改时间的，加入一条空行，也会表示内容进行更新了，缓存也会失效的。
  2. 它的修改是对于秒来说的，只要 1 秒之内修改很多的文件，服务器是不会觉得修改过文件的，这样就会继续走缓存。

ETag：

  1. 是针对于内容来说的，只要内容不更新，就会走缓存.

性能上 Last-Modified 只需要记录一个时间点。 ETag 要根据所有的内容生成具体内容的哈希值。

# 缓存的位置

当被强缓存或协商缓存命中，就会返回 304，然后直接从缓存中获取。

  1. Service Worker
  2. Memory Cache
  3. Disk Cache
  4. Push Cache

# Service Worker

Service Worker 借鉴了 Web Worker ，能够让 JS 运行在主线程之外，由于它脱离了浏览器窗口，不能够操作 DOM，依然可以进行离线缓存，消息推送，网络代理。

离线缓存 就是 Service Worker Cache.

# Memory Cache

是指内存缓存，从效率上来说是最快的，但是存活时间短，当渲染进程结束之后，内存缓存就会消失。

# Disk Cache

是指存储在磁盘中的缓存，从效率上来说是慢于内存缓存的，它的优势是它的存储时长和存储大小。

# 浏览器如何决定将资源放入 Memory Cache 还是 Disk Cache 缓存?

  1. 比较大的 JS、CSS 直接丢入磁盘，否则放入内存。
  2. 内存使用率较高时，会优先进入磁盘。

# Push Cache

是推送缓存，是 HTTP/2 中的内容。
